  assets/             (images + landing.css)
  2nd-pages/          (all lane pages)
  .nojekyll           (tells GitHub not to strip anything)

WHAT STAYS LOCAL (already in .gitignore)
  archive/
  export/
  backups, old index files, Kaveman Ops Home/

GITHUB STEPS (first time)
  1. Create a new repo on GitHub (public is fine for free Pages)
  2. In this folder, open terminal:
       git init
       git add index.html assets 2nd-pages .nojekyll .gitignore
       git commit -m "NWB Vending home page"
       git branch -M main
       git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
       git push -u origin main
  3. On GitHub: repo Settings → Pages → Source: Deploy from branch → main → / (root) → Save
  4. Wait 1–2 minutes. Site URL: https://YOUR_USERNAME.github.io/YOUR_REPO/

CUSTOM DOMAIN (NWB-Vending.com)
  1. In repo Settings → Pages → Custom domain → enter NWB-Vending.com
  2. At your domain registrar, add DNS records GitHub shows you (usually A records + CNAME for www)
  3. GitHub will add a CNAME file to the repo when you save the domain

REV CONTROL (simple)
  When Grok edits in this folder and pushes:
    1. Your LOCAL files update first (that's what we edit)
    2. git push copies that same stuff to GitHub
    3. GitHub Pages updates the live site (~1-2 min)

  So: this folder = what's live. No extra download step after a Grok session.

  Open files from HERE:
    D:\GrokBuild\experiments\Vending Machine\NWB CORE\index.html

  End of session check (Grok runs this for you):
    git status  →  should say "up to date with origin/main"

  If local ever looks old:
    - Wrong folder open (check path above)
    - Browser cache on nwb-vending.com (hard refresh: Ctrl+F5)
    - You edited on another PC → run: git pull

  Untracked files (wav, new images) stay local until you say add them.
